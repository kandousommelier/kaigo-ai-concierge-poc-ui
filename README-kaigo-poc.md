# 介護現場AIコンシェルジュ 初期PoC 手順書

この文書は、介護現場AIコンシェルジュを初期PoCとしてデプロイし、少人数の管理者・推進リーダーが共通アカウントで試すための手順です。

実際のID、パスワード、APIキー、AWS認証情報、施設名、利用者名、職員名、会話ログは、このリポジトリに記載しないでください。

## 初期PoCの範囲

- genai-web の汎用チャットのみ使う
- genai-ai-api は使わない
- RAG は使わない
- ファイル添付は使わない
- AIアプリ一覧は前面に出さない
- Cognito の標準認証を使う
- 共通アカウントは AWS 側で作成し、限定関係者にだけ共有する

## ダミー環境変数

`.env.example` にはダミー値だけを置きます。

```env
LOGIN_ID=your-login-id
LOGIN_PASSWORD=your-login-password
```

実際の値は `.env`、README、画面キャプチャ、GitHub Issues、Pull Request、コメントに書かないでください。

## デプロイ前の準備

1. AWSアカウントを用意する
2. Amazon Bedrock の利用モデルを有効化する
3. AWS CLI を設定する
4. Node.js と npm を用意する
5. AWS CDK を用意する
6. 初回のみ CDK Bootstrap を実行する

```bash
npm ci
npm -w packages/cdk run cdk -- bootstrap
```

## AWSへデプロイする

PoC用設定は `-kaigoPoc` として登録しています。

```bash
npm -w packages/cdk run cdk -- deploy --all --require-approval never -c env=-kaigoPoc
```

デプロイ後、CloudFormation Outputs または AWSコンソールで CloudFront URL を確認します。

## CognitoでPoC用共通アカウントを作る

1. AWSコンソールで Cognito を開く
2. デプロイされた User Pool を開く
3. ユーザーを作成する
4. Cognito がメール形式を求める場合は、個人名・施設名を含まないPoC専用の共通ログインIDとして管理する
5. 実際のID・パスワードは非公開の場所で管理する
6. 作成したユーザーを `UserGroup` に追加する
7. 初回ログインが必要な場合は、初回パスワード変更を行う
8. 共有前に、管理者が一度ログイン確認する

注意:

- 共通アカウントは本格的な本人認証ではありません
- 誰がどの相談をしたかは厳密には分かりません
- 会話履歴が共有される可能性があります
- 本番運用では個別アカウント、SSO、多要素認証を検討してください

## デプロイ後の確認

### URL確認

- CloudFront URL を開く
- ログイン画面が表示される
- サービス名が「介護現場AIコンシェルジュ」になっている

### ログイン確認

- 共通アカウントでログインできる
- セルフサインアップが表示されない
- ログイン画面に注意事項が表示される

### トップ画面確認

- 介護現場向けの説明が表示される
- 相談例が表示される
- 入力してはいけないことが表示される
- 回答は参考情報であることが表示される
- 初回表示時に注意事項ダイアログが表示される

### チャット確認

- チャット画面に注意事項が常時表示される
- 日本語で相談できる
- 回答が平易な日本語になっている
- 個人情報や介護記録を入れないよう促す
- ファイル添付ボタンが表示されない
- モデル選択やシステムプロンプト編集が表示されない

### スマホ表示確認

- 375px幅でログイン画面、トップ画面、注意事項、チャット画面を確認する
- 768px幅でタブレット表示を確認する
- メニューにAIアプリ一覧や利用履歴が出ていない
- 入力欄と送信ボタンが押しやすい
- 注意事項ダイアログがスクロールできる

## 施設へ案内する手順

1. CloudFront URL またはカスタムドメインURLを非公開の連絡手段で共有する
2. ID とパスワードはURLとは別の連絡手段で共有する
3. 共有先は管理者・推進リーダーなど限定関係者に絞る
4. 共有時に、以下を必ず伝える

共有文の例:

```text
介護現場AIコンシェルジュの初期PoC環境です。
介護現場の生産性向上、ICT活用、業務改善について相談・壁打ちできます。

利用者名、職員名、個別ケース、介護記録、申し送り、事故報告書等の原文は入力しないでください。
AIの回答は参考情報です。最終判断は、責任者・専門家・関係機関に確認してください。
ID・パスワードは関係者以外に共有しないでください。
```

実際のURL、ID、パスワードはこの文書に追記しないでください。

## GitHub公開前チェック

公開前に以下を確認してください。

```bash
git status --short
rg -n "AKIA|SECRET|PASSWORD|LOGIN_PASSWORD|LOGIN_ID|BEGIN PRIVATE KEY|aws_secret|aws_access|施設名|職員名|利用者名" .
```

確認事項:

- `.env` がコミット対象に入っていない
- 実際のID・パスワードが入っていない
- AWS認証情報が入っていない
- 施設名・職員名・利用者名が入っていない
- 会話ログや画面キャプチャに秘密情報が入っていない
